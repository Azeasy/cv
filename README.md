## Repo structure

```
latex/
  main.tex              # EN source (canonical)
  main_ru.tex           # RU source (manually maintained, never auto-overwritten)
  main.generated.tex    # RU preview generated by translate script (gitignored)
scripts/
  bootstrap_tm.py       # One-off seeding of TM from existing main_ru.tex
  translate_segments.py # Translation memory script
  translations_ru.json  # Translation memory (committed, manually editable)
.github/workflows/
  build-deploy.yml      # Build + deploy to Pages on push to main
  release.yml           # Build + publish GitHub Release on tag v*
  translate.yml         # Manual translation preview (workflow_dispatch)
```

PDFs are never committed. They are built by CI and either deployed to Pages or uploaded as release assets.

## Build locally

### Prerequisites

```bash
# macOS
brew install --cask mactex   # or: brew install basictex latexmk

# Ubuntu / Debian
sudo apt-get install texlive-latex-base texlive-latex-extra \
    texlive-fonts-recommended texlive-fonts-extra \
    texlive-lang-cyrillic latexmk
```

### Compile

```bash
# English CV → build/main.pdf
latexmk -pdf -interaction=nonstopmode -output-directory=build latex/main.tex

# Russian CV → build/main_ru.pdf
latexmk -pdf -interaction=nonstopmode -output-directory=build latex/main_ru.tex

# Clean auxiliary files
latexmk -C -output-directory=build latex/main.tex
```

## Publish the latest CV (Pages)

Push any commit to `main`:

```bash
git add latex/main.tex   # or main_ru.tex
git commit -m "update cv"
git push origin main
```

The `build-deploy` workflow runs automatically and deploys to GitHub Pages within ~2 minutes.

> **One-time setup:** Go to **Settings → Pages → Source** and set it to **GitHub Actions**.

---

## Publish a versioned release

Create and push a version tag:

```bash
git tag v2026.03.04
git push origin v2026.03.04
```

The `release` workflow builds both PDFs and creates (or updates) a GitHub Release marked as latest, with `cv.pdf` and `cv_ru.pdf` as downloadable assets.

Tag format convention: `v<YYYY>.<MM>.<DD>` (e.g. `v2026.03.04`).

## Translation memory

### How it works

1. `scripts/translate_segments.py` reads `latex/main.tex` and extracts translatable segments:
   - content inside `\resumeItem{...}`
   - the summary/profile paragraph
2. Each segment is **normalised** (lowercased, whitespace collapsed, punctuation stripped) and hashed via `sha1[:12]` to produce a stable ID.
3. The ID is looked up in `scripts/translations_ru.json` (the *translation memory*).
   - If found → the stored Russian translation is reused, no API call.
   - If missing → DeepL is called, the result is stored in the TM.
4. A preview file `latex/main.generated.tex` is written with all segments replaced by their Russian translations.
5. `latex/main_ru.tex` is **never touched**.

Because the ID is derived from normalised text, minor punctuation changes (adding/removing commas, changing dashes) do not invalidate existing translations.

### Run locally

```bash
export DEEPL_API_KEY=your_key_here
python scripts/translate_segments.py

# Re-translate everything (overwrite TM entries)
python scripts/translate_segments.py --force

# Use OpenAI instead of DeepL
export OPENAI_API_KEY=your_key_here
python scripts/translate_segments.py --backend openai
```

### Run via GitHub Actions

1. Add `DEEPL_API_KEY` as a repository secret (**Settings → Secrets and variables → Actions → New repository secret**).
2. Go to **Actions → Generate RU Translation Preview → Run workflow**.
3. Download the `translation-preview` artifact (contains `latex/main.generated.tex` and `scripts/translations_ru.json`).
4. Review the generated file. Copy any improvements into `latex/main_ru.tex` manually.

The workflow **never commits or pushes** anything automatically.

### Manually improving RU translations

Edit `scripts/translations_ru.json` directly:

```json
{
  "a13c92f4b21c": {
    "en": "Designed and optimized search system for legacy database design",
    "ru": "Спроектировал и оптимизировал систему поиска для устаревшей архитектуры базы данных",
    "source": "deepl",
    "updated_at": "2026-03-04"
  }
}
```

Change the `"ru"` value to your preferred translation. The script will never overwrite an existing key unless you run with `--force`.

Commit the updated `translations_ru.json` so improvements persist.
